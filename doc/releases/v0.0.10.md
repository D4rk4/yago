## Abstract

YaGo v0.0.10 aligns analyzer-equivalent word forms across local retrieval, final lexical ranking, retained search sessions, and HTML highlighting. Local results carry validated match offsets from the indexed language analyzer; peer, web, and legacy RWI rows share a bounded Unicode structural fallback where analyzer positions do not exist. The tag-driven pipeline builds and validates both product containers. A factual correction dated 2026-07-16 records an immutable-source rebuild, separate native validation, and exact-artifact GHCR publication with signed evidence; the tag, source, package assets, runtime behavior, and wire shapes remain unchanged. No setting, storage migration, listener, runtime dependency, document read, or network fan-out changes. Rollback to v0.0.9 requires no data conversion.

## Changes

### Analyzer-aligned local evidence

- Each local Bleve result carries byte offsets obtained by applying its indexed language analyzer to the final bounded snippet.
- Query targets are prepared once per result analyzer and reused across the bounded candidate window. Only the final snippet is analyzed for rendering offsets; the search path performs no additional document read.
- Analyzer evidence is authoritative. A non-empty offset set marks analyzer-equivalent forms, while an analyzed no-match remains distinct from an unavailable analyzer and does not fall through to a heuristic match.
- Offsets survive result mapping, page-evidence replacement, content-cluster selection, retained paging sessions, and session cloning.

### Federated lexical evidence and ranking

- Peer, web, and legacy RWI rows have no trustworthy analyzer positions, so their visible title and snippet use one bounded Unicode structural matcher for coverage, ordered proximity, unordered proximity, and fallback highlighting.
- The matcher preserves combining sequences, requires bounded token tails, retains boundary-delimited punctuation-bearing identifiers, and permits intra-token evidence for scripts that do not conventionally delimit every word with whitespace.
- The peer profile scorer prepares the title and decoded URL evidence once per row. Web URLs remain admission evidence and do not become a second final-ranking field.
- Analyzer-equivalent local positions and bounded federated fallback positions now reach the same lexical-gap and proximity calculations instead of losing evidence after retrieval.

### Safe human-facing highlighting

- The public portal, Admin search, and YaCy HTML render only validated query-match spans after escaping the complete snippet.
- Overlapping spans are coalesced, malformed byte ranges are discarded, and every accepted boundary is checked against UTF-8 rune boundaries before markup is inserted.
- Russian sibling forms such as `полномочия`, `полномочий`, and `полномочиями` therefore retain the same local retrieval evidence, ranking treatment, and visible marking.

### Documentation maintenance

- The committed v0.0.9 engineering memo now uses trademark-neutral wording for the Admin login composition. The historical tag and released artifacts remain unchanged.
- The feature catalog, project overview, search specification, and YaGoRank operator documentation describe the analyzer-offset path and the bounded federated fallback.

### Release container gate

- A blocking native Linux matrix builds the node and crawler containers on amd64 and arm64 after `make verify` succeeds.
- Each build receives the complete release tag as `VERSION` and the tagged commit as `SOURCE_REVISION`. The gate checks Docker's reported architecture, the exact OCI source and revision labels, both binary version strings, and the crawler image's bundled Firefox executable.
- Pinned Trivy 0.72.0 scans each local image for vulnerabilities, secrets, and image misconfigurations. A HIGH or CRITICAL finding stops release publication.
- Package construction and container validation ran independently, and the release job required both. The initial tag run kept images inside isolated runners; the 2026-07-16 factual correction records a controlled later rebuild from the immutable release source and promotion of the exact successful backfill-validation artifacts.

## Upgrade and rollback

1. Download the architecture-matching package and verify its GitHub attestation against this repository, release workflow, tag ref, and source commit:

   ```sh
   gh release download v0.0.10 --repo D4rk4/yago \
     --pattern 'yago_0.0.10_amd64.deb' --dir /tmp/yago-v0.0.10
   source_digest=$(gh api repos/D4rk4/yago/commits/v0.0.10 --jq .sha)
   gh attestation verify /tmp/yago-v0.0.10/yago_0.0.10_amd64.deb \
     --repo D4rk4/yago \
     --signer-workflow D4rk4/yago/.github/workflows/release.yml \
     --source-ref refs/tags/v0.0.10 \
     --source-digest "$source_digest"
   ```

2. Take a consistent backup of `/opt/yago/data` on a filesystem with sufficient free space.
3. Stop both services:

   ```sh
   sudo systemctl stop yagocrawler yago-node
   ```

4. Install the verified published package:

   ```sh
   sudo dpkg -i /tmp/yago-v0.0.10/yago_0.0.10_amd64.deb
   ```

5. Start the node, verify health and readiness, then start the crawler:

   ```sh
   sudo systemctl start yago-node
   wget -qO- http://127.0.0.1:9090/health
   wget -qO- http://127.0.0.1:9090/ready
   sudo systemctl start yagocrawler
   systemctl --no-pager --full status yago-node yagocrawler
   ```

The package enables but does not start the units. No persistent format changes in this release require conversion during rollback. To return to v0.0.9, stop both services, install the published v0.0.9 package, start the node and verify readiness, then start the crawler. Restore the stopped pre-upgrade backup when exact pre-upgrade state is required or if an upgrade health check failed after data was written.

## Known limitations

- Analyzer offsets cover the bounded final snippet rather than an arbitrary full-document highlighting surface.
- Multiword YaCy swarm retrieval still hashes exact RWI surface forms. This release aligns evidence after a remote row arrives but does not multiply peer fan-out across morphology variants; a peer-only sibling inflection can therefore remain undiscovered for a multiword query.
- The peer, web, and legacy RWI matcher is a bounded structural fallback, not a complete language-specific morphology analyzer. Local Bleve results remain the authoritative analyzer-backed path.
- The local CJK index remains bigram-based. This release does not add dictionary segmentation, script conversion, or a unigram index.
- External web-provider availability is not guaranteed. Only operator-permitted results completing inside the existing fixed search budget can contribute.
- This release does not add a persistent cross-node query-match-offset protocol; federated rows continue to derive evidence from their bounded visible fields.
- Factual correction 2026-07-16: release containers are now published only under immutable `v0.0.10` GHCR tags with signed evidence. No `latest`, major-only, minor-only, branch, or date alias exists; the GitHub Release continues to contain only its six Debian, RPM, and tarball assets.
