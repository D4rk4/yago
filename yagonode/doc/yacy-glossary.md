# YaCy glossary

Verbatim definitions from YaCy's own documentation, for grounding the domain model
independently of this project's implementation. Sources are listed at the end.

## Core index concepts

- **Reverse Word Index (RWI)** — "Indexing is the process of creation a Reverse Word
  Index (RWI) datastructure from a given set of text documents. It means that a
  document-words relation is reversed to a word-documents relation." "In this index,
  urls of pages containing that particular keyword are stored for every keyword."
- **Word Index** — "Reverse Word Index. This is generated by the indexer from the
  collected data and stored in the database."
- **DHT (Distributed Hash Table)** — "RWI is distributed over [the] whole network …
  creating the global Distributed Hash Table (DHT). The DHT structures the network as
  a virtual ring, where peer hashes determine proximity, allowing efficient formation
  of connections by routing queries to nearby nodes based on hash values." A DHT stores
  (key, value) pairs that any participating node can efficiently retrieve.
- **DHT distribution** — "The RWI is the distributed Word Index that is generated and
  then waits to be distributed to other peers according to a distributed hashtable
  schema." "The DHT target (another peer) is selected … then index entries are selected
  for transfer. The rule is, that exactly those index entries are selected for transfer
  which I would expect to be at the peer when I search using the same DHT rule."
- **Redundancy / replication** — "How many identical copies of your RWI are sent to
  other hosts, before they are deleted locally." Defaults: three copies for senior
  peers, one for junior peers.

## Network / peer concepts

- **Peer** — "A peer is a communications endpoint in a computer network. Each peer
  offers its services and uses the services of other peers." In YaCy a peer provides
  web-indexing services to other peers on the search network.
- **Senior peer** — a peer with network contact reachable by others; can search the
  local and global index.
- **Junior peer** — a peer connected to the network but unreachable by others (e.g.
  firewalled); "can contribute to the network by submitting index files."
- **Principal peer** — "a Senior peer that uploaded an additional peer-list to a server."
- **Seed / seed-list** — hard-coded bootstrap servers that provide an initial peer list
  (IP addresses, port numbers, peer hashes) to newly joining peers.

## Hash notes

- A **peer hash** determines a peer's position on the DHT ring (proximity / routing).
- A **word hash** is the DHT key derived from a keyword; it selects which peer(s) should
  hold that word's postings (same rule used for store and for search).
- A **URL hash** identifies an indexed URL; URL metadata is stored alongside the postings
  that reference it.
